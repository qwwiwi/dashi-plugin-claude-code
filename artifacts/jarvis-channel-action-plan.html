Линия A · Channel migration

Phase 1 Прототип channel-плагина · пилот Сильваны ≈ 2 ч
Минимальный jarvis-channel.ts: один бот (@fridayhumanbot), allowlist, push event в running session, reply tool. E2E smoke с Telegram.

Шаги

Создать репо qwwiwi/jarvis-channel (private)
gh repo create qwwiwi/jarvis-channel --private. Зарегистрировать в mapping-repositories.
Init Bun проект + MCP SDK
bun init -y && bun add @modelcontextprotocol/sdk zod
Написать jarvis-channel.ts (минимум)
MCP server со stdio + claude/channel capability + reply tool. Telegram long polling getUpdates per bot token из env.
Создать plugin.json + локальный marketplace
/plugin marketplace add ~/jarvis-channel → /plugin install jarvis-channel@local
launchd unit ai.orgrimmar.silvana-channel.plist
claude --channels plugin:jarvis-channel@local --workspace ~/.claude-lab/silvana/.claude --dangerously-skip-permissions --dangerously-load-development-channels
E2E smoke
Принц пишет «привет» в @fridayhumanbot → Сильвана отвечает через channel. Латентность ≤ 5 сек.

Готово, когда

Сильвана отвечает на сообщение в Telegram, расход в Max-пуле (не в SDK), gateway.py продолжает работать параллельно как fallback для остальных агентов.

Phase 2 Портация бизнес-логики из gateway.py ≈ 3 ч
Перенести всю known-good логику Jarvis в TypeScript-плагин. После — функциональный паритет.

Что портируем

Media-group buffer (2-сек окно)
Из gateway.py:1932 — склейка альбомов фото/видео. Setimeout / setInterval pattern в TypeScript.
Reply-injection
parsing reply_to_message → untrusted metadata block + JSON. Кладём в meta.reply_context.
Photo download → local path
Bot API getFile → fetch → /tmp/jarvis-photos/{file_id}.jpg → path в meta.
Voice transcription
Если есть Groq Whisper в gateway.py — переносим pre-emit transcribe → текст в content.
HTML formatting + split ≤ 3500 chars
В reply tool: автоматический split на куски, telegram HTML escape для опасных символов.
Numeric ID allowlist
Конфиг allowlist.json с user_ids. Privacy ON для групповых чатов.
Phase 3 Permission relay (бонус-фича Anthropic) ≈ 1 ч
Опасные операции (rm -rf, sudo, git push --force) спрашивают подтверждение у принца через Telegram. Ответ «yes abcde / no abcde».

Шаги

Добавить capability claude/channel/permission
В Server constructor. Анонсирует что плагин принимает permission_request.
Handler notifications/claude/channel/permission_request
Format prompt: «Silvana хочет run Bash: rm /tmp/x. Ответь yes abcde / no abcde» в Telegram.
Parse inbound «yes/no abcde»
Regex /^\\s*(y|yes|n|no)\\s+([a-km-z]{5})\\s*$/i. Эмит permission notification вместо обычного channel event.
Убрать --dangerously-skip-permissions
Теперь не нужно — permission relay покрывает unattended случай через Telegram.
Phase 4 Rollout остальных 5 агентов ≈ 2 ч
Сильвана работает на Channel — теперь переезжают Кельтас, Гаррош, Иллидан (Mac mini), Тралл, Артас (Thrall VPS).

На каждого агента

launchd unit (Mac mini) или systemd unit (Thrall)
Шаблон от Сильваны. Подменить workspace path + bot token env.
Allowlist в config (только принц)
numeric ID 164795011 для всех агентов.
Health monitoring (Артас)
Артас heartbeat каждые 15 мин → если session down → alert принцу.
RAM check
6 параллельных claude sessions × ~500 MB = 3 GB. На Mac mini (16 GB) + Thrall (8 GB) — терпимо.
Phase 5 Decommission gateway.py ~ 1 неделя
Старый Python daemon переходит в standby, затем отключается. Только после неделю стабильной работы Channel-архитектуры.

Шаги (неделя ожидания)

Сразу: gateway.py в standby, не отвечает на Telegram
launchctl unload ~/Library/LaunchAgents/ai.orgrimmar.gateway.plist. Бинарь и конфиг остаются.
Неделя observation
Дневной отчёт от Артаса: были ли inсиденты, latency, OOM, compact-проблемы.
Если стабильно — финальное удаление
Архив gateway.py в репо jarvis-telegram-gateway/legacy/. Удалить launchd plist. Документация для rollback.
Если проблемы — откат за 5 минут
launchctl load ai.orgrimmar.gateway.plist + disable channel units. Возвращается старая архитектура.

Перенести один Anthropic-зависимый cron как пилот. Доказать что MiniMax справляется с типичной задачей и Anthropic-расход уходит.

Действия

Список Anthropic-зависимых cron'ов
Записать в core/projects/jarvis-channel-migration/cron-inventory.md с маркером MIGRATE / KEEP.
По каждому MIGRATE → переезд на Тиранду
Pattern: pull data → analyze on MiniMax → если нужен Opus → mention агента в Orgrimmar Ops group.
KEEP не трогаем
Vault autosync, cognify, gbrain audit, restic backups, log rotate, telegram-chip — продолжают работать как есть.
